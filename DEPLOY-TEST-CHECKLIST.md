## 🎨 Phase 3 Stage 1-2 애니메이션 테스트 (10분)

### Stage 1: 접근성 & 기본 UX

#### ✅ 1. prefers-reduced-motion (접근성)
**테스트 방법**:
1. Chrome DevTools (F12) → More tools → Rendering
2. "Emulate CSS media feature prefers-reduced-motion" → reduce 선택
3. 페이지 새로고침
4. 모든 애니메이션이 즉시 완료되는지 확인 (0.01ms)

**예상 결과**: 모달, 토스트, 버튼이 애니메이션 없이 즉시 나타남

- [ ] 모달 애니메이션 비활성화 확인
- [ ] 토스트 애니메이션 비활성화 확인
- [ ] 버튼 호버 애니메이션 비활성화 확인

---

#### ✅ 2. focus-visible (키보드 네비게이션)
**테스트 방법**:
1. Tab 키로 포커스 이동
2. 모든 버튼/입력창에 초록색 포커스 링(3px) 표시 확인

**예상 결과**: 초록색 포커스 링이 명확하게 보임

- [ ] 검색 입력창 포커스 링 확인
- [ ] GPS 버튼 포커스 링 확인
- [ ] 필터 탭 포커스 링 확인
- [ ] 파트너 카드 포커스 링 확인
- [ ] 모달 내부 버튼 포커스 링 확인

---

#### ✅ 3. 모달 Fade In/Out
**테스트 방법**:
1. 파트너 카드 클릭 → 모달 열림
2. 모달이 부드럽게 나타나는지 확인 (250ms)
3. X 버튼 또는 ESC 키로 닫기
4. 모달이 부드럽게 사라지는지 확인

**예상 결과**:
- 모달 배경: opacity 0 → 1
- 모달 콘텐츠: scale 0.95 → 1 (살짝 커지면서 나타남)

- [ ] 모달 Fade In 확인 (부드러운 등장)
- [ ] 모달 Scale 애니메이션 확인 (살짝 커짐)
- [ ] 모달 Fade Out 확인 (부드러운 사라짐)

---

#### ✅ 4. 토스트 Slide In
**테스트 방법**:
1. 즐겨찾기 버튼(♥) 클릭
2. 토스트 메시지가 우측에서 슬라이드되는지 확인 (300ms)

**예상 결과**: 토스트가 우측 바깥에서 안쪽으로 슬라이드

- [ ] 토스트 Slide In 확인 (우측에서 등장)
- [ ] 토스트 자동 사라짐 확인 (3초 후)

---

#### ✅ 5. 버튼 Hover 효과
**테스트 방법**:
1. 마우스를 버튼에 올림 (Hover)
2. 버튼이 살짝 위로 이동하거나 커지는지 확인

**예상 결과**:
- 일반 버튼: translateY(-2px) 위로 이동
- 즐겨찾기 버튼: scale(1.15) 살짝 커짐

- [ ] GPS 버튼 Hover 확인 (위로 이동)
- [ ] 필터 버튼 Hover 확인
- [ ] 모달 액션 버튼 Hover 확인
- [ ] 즐겨찾기 버튼 Hover 확인 (커짐)

### Stage 2: 인터랙션 강화

#### ✅ 6. 하트 Bounce 애니메이션
**테스트 방법**:
1. 파트너 카드의 즐겨찾기 버튼(♥) 클릭
2. 하트가 통통 튀는지 확인 (500ms)

**예상 결과**:
- 하트가 커졌다(1.3) → 작아졌다(0.9) → 중간(1.15) → 원래 크기(1)로 변화
- 4단계 Bounce 애니메이션

- [ ] 리스트 카드의 하트 Bounce 확인
- [ ] 모달 내부 즐겨찾기 버튼 Bounce 확인
- [ ] 애니메이션이 500ms 동안 진행되는지 확인

---

#### ✅ 7. 카드 Slide In 애니메이션
**테스트 방법**:
1. 페이지 로드 후 파트너 리스트 확인
2. 스크롤하면서 카드가 순차적으로 나타나는지 확인

**예상 결과**:
- 카드가 아래에서 위로 슬라이드 (translateY(20px) → 0)
- 동시에 opacity 0 → 1
- Intersection Observer로 화면에 보이는 순간 애니메이션 시작

- [ ] 초기 로드 시 카드 Slide In 확인
- [ ] 스크롤 시 순차 애니메이션 확인
- [ ] 브라우저 호환성 확인 (IE는 즉시 표시)

---

#### ✅ 8. 거리 배지 Pulse 애니메이션
**테스트 방법**:
1. GPS 버튼 클릭 → 내 위치 기준으로 거리 계산
2. 파트너 카드에 표시된 거리 배지 확인
3. 배지가 맥동하는지 확인 (2초 무한 반복)

**예상 결과**:
- 거리 배지가 살짝 커졌다가(1.05) 작아지는 부드러운 맥동
- 2초 주기로 무한 반복

- [ ] 거리 배지 Pulse 확인 (맥동 효과)
- [ ] 2초 주기 확인
- [ ] 너무 과하지 않은지 확인 (scale 1.05까지만)

## 🚨 문제 발생 시 대응

### 저장 실패
**증상**: "데이터 수정 실패" 메시지
**원인**: 금지 패턴 (이모지, async/await, 템플릿 리터럴)
**해결**:
```bash
cd /Users/jangjiho/workspace/partner-map
./scripts/check-forbidden-patterns.sh
```

### 콘솔 에러
**증상**: F12 콘솔에 빨간색 에러 메시지
**대응**: 에러 메시지 스크린샷 → 개발자에게 전달

### 애니메이션 미작동
**증상**: 모달이 애니메이션 없이 즉시 나타남
**원인**: CSS 저장 실패 또는 브라우저 캐시
**해결**:
1. Ctrl+Shift+R (강제 새로고침)
2. 메이크샵 CSS 탭 확인 → 1202줄 모두 저장되었는지 확인

### 아이콘 깨짐
**증상**: ♥ 대신 □ (네모) 표시
**원인**: Phosphor Icons CDN 로드 실패
**해결**: HTML 탭 6번 줄 확인
```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@phosphor-icons/web@2.1.2/src/regular/style.css">
```
