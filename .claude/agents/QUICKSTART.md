## 5분 안에 시작하기

### 1단계: 자연어로 요청하기
특별한 명령어나 문법이 필요 없습니다. 평소에 말하듯이 요청하면 됩니다.

```
✅ 좋은 예:
"지도에 파트너 위치 마커를 표시해줘"
"파트너 검색 필터를 추가해줘"
"메이크샵에 배포 가능한지 검증해줘"

❌ 나쁜 예:
"그거 해줘"
"수정해줘"
"확인해줘"
```

### 2단계: 시스템이 자동으로 처리
키워드를 분석해서 적합한 에이전트를 자동으로 선택합니다.

```
요청: "지도 마커 클릭 시 파트너 정보 표시"
↓
자동 분석: "지도", "마커", "클릭" → map-engineer
         "정보 표시" → frontend-engineer
↓
실행: map-engineer + frontend-engineer 협업
```

### 3단계: 결과 확인 및 피드백
결과물을 확인하고 추가 요청이 있으면 다시 자연어로 말하면 됩니다.

```
"잘됐어! 그런데 마커 색상을 브랜드 컬러로 바꿔줘"
→ graphic-designer가 자동으로 호출됨
```

## 3가지 기본 패턴

### 패턴 1: 단일 에이전트 (가장 흔함)
**언제 사용**: 명확한 단일 작업

```
📝 예시: "파트너 목록을 카드 형태로 디자인해줘"

🤖 처리:
- 키워드: "디자인", "카드 형태"
- 호출: ui-designer
- 소요 시간: 1-2분

💡 팁: 도메인을 명확하게 언급하면 더 정확함
```

### 패턴 2: 병렬 실행 (시간 단축)
**언제 사용**: 독립적인 여러 작업을 동시 수행

```
📝 예시: "배포 전 최종 점검해줘"

🤖 처리:
- 키워드: "배포", "최종 점검"
- 호출: makeshop-specialist + code-reviewer + security-auditor (동시)
- 소요 시간: 2-3분 (순차 실행 대비 60% 단축)

💡 팁: "동시에", "병렬로" 키워드를 사용하면 강제 병렬 실행
```

### 패턴 3: 순차 실행 (단계별 진행)
**언제 사용**: 이전 단계의 결과가 다음 단계의 입력이 되는 경우

```
📝 예시: "파트너 즐겨찾기 기능을 추가해줘"

🤖 처리:
- 1단계: product-manager (요구사항 정의)
- 2단계: frontend-engineer (UI 구현)
- 3단계: qa-engineer (테스트)
- 4단계: technical-writer (문서화)
- 소요 시간: 5-7분

💡 팁: "단계별로", "순서대로" 키워드 사용
```

## 5가지 실전 예시

### 예시 1: 간단한 UI 수정 (초급)
```
요청: "검색 버튼 색상을 파란색으로 바꿔줘"

자동 처리:
- ui-designer 호출
- CSS 수정 (:root 변수 활용)
- 반응형 확인

결과:
.search-button {
  background-color: var(--pm-primary);  /* #2196F3 */
}

소요 시간: 1분
```

### 예시 2: 이모지 대체 작업 (중급)
```
요청: "파트너 목록의 이모지를 메이크샵 호환 아이콘으로 바꿔줘"

자동 처리:
1. graphic-designer: 이모지 분석 및 매핑표 생성
   📍 → <i class="ph ph-map-pin"></i>
   📞 → <i class="ph ph-phone"></i>
   ❤️ → <i class="ph ph-heart"></i>

2. makeshop-specialist: 이모지 제거 + Phosphor 클래스 적용

3. frontend-engineer: 아이콘 스타일 통합

4. qa-engineer: 메이크샵 에디터 저장 테스트

결과: 메이크샵 호환 코드, 저장 가능

소요 시간: 3-4분
```

### 예시 3: 새 기능 추가 (중급)
```
요청: "즐겨찾기한 파트너만 지도에 표시하는 필터를 추가해줘"

자동 처리:
1. product-manager: 요구사항 정의
   - 사용자 스토리
   - 수용 기준

2. retail-expert: 필터 로직 설계

3. ui-designer: 필터 UI 디자인

4. frontend-engineer: 구현
   - localStorage에 즐겨찾기 저장
   - 필터 체크박스 추가
   - 마커 표시/숨김 로직

5. qa-engineer: 테스트
   - 필터 ON/OFF 시나리오
   - localStorage 저장 확인

6. technical-writer: MEMORY.md 업데이트

결과: 완전히 동작하는 필터 기능

소요 시간: 7-10분
```

### 예시 4: 배포 준비 (중급)
```
요청: "현재 코드를 메이크샵에 배포해도 될지 검증해줘"

자동 처리 (병렬):
1. makeshop-specialist:
   ✅ 템플릿 리터럴 이스케이프 확인
   ✅ 이모지 제거 확인
   ✅ 파일 크기 30KB 이하
   ❌ HTML 문자열 연결 15줄 (10줄 이하 권장)

2. code-reviewer:
   ✅ IIFE 패턴 적용
   ✅ 이벤트 위임 사용
   ⚠️ 변수명 일관성 개선 제안

3. security-auditor:
   ✅ XSS 방어
   ✅ API 키 보안
   ⚠️ localStorage 암호화 권장

결과:
- 배포 가능 여부: ⚠️ 수정 필요 (HTML 문자열 분할)
- 수정 사항: 1건 (필수)
- 개선 제안: 2건 (선택)

소요 시간: 2-3분
```

### 예시 5: 긴급 디버깅 (고급)
```
요청: "배포 후 지도가 안 뜨는데 원인 찾아서 수정해줘"

자동 처리 (순차):
1. map-engineer: 진단
   - 브라우저 콘솔 확인
   - 카카오맵 API 로드 상태 확인
   - 발견: API 키 누락

2. makeshop-specialist: 제약사항 확인
   - 스크립트 로드 순서 검증
   - 발견: 스크립트 순서 문제

3. map-engineer: 수정
   - API 키 추가
   - 스크립트 순서 조정
   - 초기화 코드를 onload 이벤트로 래핑

4. code-reviewer + qa-engineer: 검증 (병렬)
   - 코드 리뷰
   - 실제 환경 테스트
   - 결과: ✅ 정상 동작

결과:
- 수정된 코드
- 트러블슈팅 가이드 업데이트

소요 시간: 5-7분
```

## 주의사항

### ⚠️ 하지 말아야 할 것

#### 1. 모호한 요청
```
❌ "고쳐줘"
❌ "좀 더 좋게"
❌ "그거 해줘"

✅ "검색 버튼의 배경색을 #2196F3으로 변경해줘"
✅ "파트너 목록에 카드 호버 효과를 추가해줘"
```

#### 2. 과도한 세부 사항
```
❌ "ui-designer에게 CSS 파일을 열어서 .search-button 클래스의 background-color 속성을 #2196F3으로 변경하라고 전달해줘"

✅ "검색 버튼을 파란색으로 바꿔줘"
(시스템이 자동으로 ui-designer 호출)
```

#### 3. 불필요한 검증 요청
```
❌ "버튼 텍스트만 변경하는데 보안 감사 포함해서 전체 검증해줘"
(과도한 리소스 낭비)

✅ "버튼 텍스트 '확인' → '저장' 변경 (간단한 작업)"
```

### 🎯 해야 할 것

#### 1. 도메인 키워드 사용
```
✅ "지도 마커 클러스터링 추가"
→ map-engineer 자동 호출

✅ "Phosphor Icons로 아이콘 통일"
→ graphic-designer 자동 호출
```

#### 2. 범위 명시
```
✅ "파트너 검색 필터만 추가 (배포는 나중에)"
✅ "배포 가능 여부만 확인 (수정은 내가 할게)"
```

#### 3. 우선순위 표시
```
✅ "긴급: 배포 후 지도 오류 수정"
✅ "낮은 우선순위: 검색 성능 최적화"
```

## 자주 묻는 질문 (FAQ)

### Q1: 에이전트를 직접 지정할 수 있나요?
```
A: 네, 가능합니다.

"@graphic-designer: 파트너맵 로고 디자인해줘"
"@makeshop-specialist: 현재 코드 검증해줘"
```

### Q2: 여러 에이전트를 동시에 호출할 수 있나요?
```
A: 네, 명시적으로 지정하면 됩니다.

"@graphic-designer와 @ui-designer가 협업해서 디자인 시스템 구축해줘"
```

### Q3: 체이닝 순서를 직접 지정할 수 있나요?
```
A: 네, 단계별로 요청하면 됩니다.

"1단계: graphic-designer가 아이콘 디자인
 2단계: makeshop-specialist가 메이크샵 호환 코드로 변환
 3단계: qa-engineer가 테스트"
```

### Q4: 이전 작업 결과를 재사용할 수 있나요?
```
A: 네, 컨텍스트를 명시하면 됩니다.

"이전에 만든 아이콘 세트를 재사용해서 새 페이지에 적용해줘"
```

### Q5: 작업을 중간에 멈출 수 있나요?
```
A: 네, 언제든지 피드백을 주면 됩니다.

작업 중: "잠깐, 여기까지만 하고 내가 확인할게"
작업 후: "이건 아닌 것 같아. 다른 방식으로 다시 해줘"
```

## 효율적인 요청 패턴

### 패턴 A: 점진적 개선
```
1차 요청: "파트너 검색 기능 최소 버전 구현"
→ 결과 확인

2차 요청: "검색어 자동완성 추가"
→ 결과 확인

3차 요청: "검색 성능 최적화 (debounce)"
→ 완료
```

### 패턴 B: 병렬 작업
```
"다음 3개 작업을 동시에 진행해줘:
1. 로고 디자인
2. 검색 필터 구현
3. 성능 최적화"

→ 시간 절약 (순차 대비 60%)
```

### 패턴 C: 조건부 실행
```
"코드 리뷰해서 문제가 있으면 수정, 없으면 바로 배포"
→ 불필요한 단계 생략
```
